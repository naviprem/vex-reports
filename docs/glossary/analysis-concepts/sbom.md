# SBOM (Software Bill of Materials)

The machine-readable inventory of every [[package]] inside an artifact, with versions and identifiers ([[purl|PURLs]]/[[cpe|CPEs]]). Standard formats: SPDX and CycloneDX; generated by tools like [[syft]].

An SBOM is a *claim about evidence found*, not ground truth — [[vendoring|vendored]] code is invisible to it. "No findings" ≠ "no vulnerabilities".
